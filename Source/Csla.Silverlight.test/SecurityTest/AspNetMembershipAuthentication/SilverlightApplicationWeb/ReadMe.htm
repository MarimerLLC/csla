Running Tests in this project

All of the tests in this solution should be located in SilvelightApplication project, as a separate TestClasses (SecurityTests for example).  Silvelight ApplicationWeb is a web project that contains a page called SilverlightApplicationTestPage.aspx. This page is a container for Silverlight code, and should be a startup page for this site.

Since the tests have to be run inside Silverlight runtime, we use our own UnitDriven test runner, which is a UI that enumerates and runs automated tests in Silverlight runtime. Therefore running this site will bring up UI that allows us to run automated tests.

Configuration

Since these are a set of Authentication/AUthorization tests on top of Membership API following is needed and is utilized in these tests:

  • Membership API web configuration for Membership/Role providers
  • Mock Membership Provider and Mock Role provider classes that return pre-determined authentication responses to our automated tests and are invoked through Membership API
  • Csla objects that implement MambershipIdentity (object that transfers the identity info from server to client), and implementation of BusinessBasePrincipal. Usage of these classes in the encapsulated tests should be an example for developers implementing a Membership based authentication on Silverligt applications
